# CATEGORY_MAPPING

Top-level categories were forced to exactly:

- `Educational Kits`
- `Sensory Development Kits`

## Mapping table

| Raw category label | Normalized top-level category | Normalized subcategory strategy | Notes |
|---|---|---|---|
| `Playset` | `Educational Kits` | `Playset` | Core coding product. |
| `Extra / Map` | `Sensory Development Kits` | `Transparent Map` | Tactile/open-ended map format. |
| `Extra Cards` | `Educational Kits` | `Cards` | Literacy/numeracy learning cards kept educational. |
| `Adventure / Logic` | `Educational Kits` | `Logic Pack` | Logic/story expansion content. |
| `Extra Blocks` | `Educational Kits` | `Blocks` | Coding expansion blocks. |
| `Adventure / Map` | `Sensory Development Kits` | `Adventure Map` | Themed tactile maps grouped as sensory/spatial products. |

## Preservation of lost detail
Because the top-level taxonomy was compressed into only two buckets, extra context was preserved in:
- `subcategory`
- `skills`
- `sensory_focus`
- `tags`

## Needs Review
1. Adventure maps could also reasonably be classified under `Educational Kits` because they support coding/story learning as much as sensory exploration.
2. Transparent-map accessories (alphabet/number cards) were kept under `Educational Kits` because their primary use is literacy/numeracy, even though they are physically used with a sensory map.
